Golang协程调度

go调度器的由来

单进程时代：

  • 单一执行，计算机一个任务一个任务处理

多进程操作系统：

  • 使用CPU调度器：
    • 不同的时间片ABC轮流执行
  • 多进程带来了新的问题：
    • 时间片的切换成本（保存现场）
    • 并且使得开发设计变得复杂（状态锁，同步等问题）
    • 为了保证多任务，开了很多线程，高度消耗调度CPU并且消耗内存
      • 进程占用 4GB
      • 线程占用4M

操作系统分为 用户态和内核态

如果我们让内核只看到一个线程，在线程里面开多个小线程（co-routine）协程
